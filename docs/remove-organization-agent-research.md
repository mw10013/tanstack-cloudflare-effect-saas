## Data Considerations

### Agent SQLite Data

The agent creates internal SQLite tables:

- `Upload` - classification tracking
- `GoogleConnection` - OAuth tokens
- `GoogleOAuthState` - OAuth state
- `GoogleSheetsConfig` - default spreadsheet
- `GoogleSpreadsheetCache` - cached spreadsheets

These are stored in the Durable Object's attached SQLite and will be deleted when the DO class is removed. No migration needed.

### R2 Data

Uploads stored in R2 are independent of the agent. The queue sends notifications to the agent, but if the queue consumer is removed, uploads will still work - they just won't be classified automatically.

Remove the queue and queue functionality

### D1 Data

No D1 tables are created by the agent - all data is in DO-attached SQLite.
