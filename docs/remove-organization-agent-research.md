# Removing Organization Agent Research

## Overview

This document outlines the steps required to safely and cleanly remove `@src/organization-agent.ts` and all related components from the codebase.

## Files to Remove

### Core Agent Files

| File                           | Description                                                          |
| ------------------------------ | -------------------------------------------------------------------- |
| `src/organization-agent.ts`    | Main agent class with workflows, SQLite tables, and callable methods |
| `src/organization-messages.ts` | WebSocket message schemas used by the agent                          |

### Route Files to Remove

| File                                          | Action                 |
| --------------------------------------------- | ---------------------- |
| `src/routes/app.$organizationId.agent.tsx`    | Remove + patch sidebar |
| `src/routes/app.$organizationId.chat.tsx`     | Remove + patch sidebar |
| `src/routes/app.$organizationId.upload.tsx`   | Remove + patch sidebar |
| `src/routes/app.$organizationId.google.tsx`   | Remove + patch sidebar |
| `src/routes/app.$organizationId.workflow.tsx` | Remove + patch sidebar |

### Worker Configuration

**`wrangler.jsonc`** - Remove or modify:

```diff
  "durable_objects": {
    "bindings": [
-     {
-       "name": "ORGANIZATION_AGENT",
-       "class_name": "OrganizationAgent"
-     }
    ]
  },
  "workflows": [
-   {
-     "name": "OrganizationWorkflow",
-     "binding": "OrganizationWorkflow",
-     "class_name": "OrganizationWorkflow"
-   },
-   {
-     "name": "OrganizationImageClassificationWorkflow",
-     "binding": "OrganizationImageClassificationWorkflow",
-     "class_name": "OrganizationImageClassificationWorkflow"
-   }
  ],
```

Also remove from `env.production` block.

**`worker-configuration.d.ts`** - Auto-generated by Wrangler; run `pnpm wrangler types` after updating wrangler.jsonc.

### Worker Entry Point

**`src/worker.ts`** - Modify:

1. Remove imports:
   - `extractAgentName` from `./organization-agent`
   - Export of `OrganizationAgent`, `OrganizationWorkflow`, `OrganizationImageClassificationWorkflow`

2. Remove queue handler (entire `queue` export function) - R2 upload/delete queue that calls agent stubs

3. Remove agent routing callbacks in `fetch` - the `routeAgentRequest` call and its `onBeforeConnect`/`onBeforeRequest` handlers

## Removal Order

1. **Routes**: Remove the 5 route files and patch sidebar

2. **Worker**: Update `src/worker.ts` to remove:
   - Agent imports and exports
   - Queue handler function
   - Agent route callbacks and `routeAgentRequest` call

3. **Wrangler**: Update `wrangler.jsonc` to remove:
   - DO binding
   - Workflows
   - R2, AI, R2_UPLOAD_QUEUE bindings
   - Related env vars

4. **Regenerate types**: Run `pnpm wrangler types`

5. **Typecheck**: Run `pnpm typecheck` and fix any remaining references

6. **Lint**: Run `pnpm lint`

## Data Considerations

- Agent SQLite tables are in DO-attached SQLite - will be deleted when DO removed
- R2 uploads are independent - remove queue and classification but uploads can remain if needed elsewhere
- No D1 tables created by agent

## Post-Removal Verification

After removal, verify:

1. `pnpm typecheck` passes
2. `pnpm lint` passes
3. Dev server starts without errors
4. Sidebar renders correctly
